## 6）`docs/提交规范.md`

```md
# 提交规范

## 1. 提交前必须做
1. 查看 `git status`
2. 查看 `git diff --name-only`
3. 查看 `git log --oneline -5`
4. 查看 `docs/变更记录.md`
5. 查看 `docs/实验启动记录.md`

## 2. 提交原则
- 只提交当前任务相关文件
- 不允许 `git add .`
- 必须同步提交对应文档更新
- commit message 必须用中文

## 3. 推荐提交格式
[类型] 模块：一句话说明

示例：
- [功能] 实验脚本：新增统一日志记录
- [修复] 数据路径：改为读取 paths.local.yaml
- [实验] B线最小run：补充 B0-B3 初始结果
- [文档] 主规划：更新当前阶段和下一步任务

## 4. 禁止提交内容
- 数据集
- 日志
- checkpoints
- outputs
- results
- 临时调试文件
- 实际服务器路径配置
