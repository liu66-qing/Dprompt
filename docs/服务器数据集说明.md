# 服务器数据集说明

## 1. 原则
1. 数据集只存放在服务器，不提交到 GitHub。
2. 仓库只提交路径模板，不提交真实路径。
3. 真实路径配置保存在 `configs/paths.local.yaml`。
4. 代码必须从配置文件读取数据路径，禁止把服务器路径硬编码在代码里。

## 2. 需要提交到仓库的文件
- `configs/paths.example.yaml`

## 3. 禁止提交到仓库的文件
- `configs/paths.local.yaml`

## 4. 推荐使用方式
运行训练/测试脚本时，统一传入：
```bash
--data_cfg configs/paths.local.yaml
